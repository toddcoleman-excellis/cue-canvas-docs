Progress Indicator

A slim 4px progress bar used inside toast notifications to communicate file export, upload, or other background task status. Comes in two color variants — blue for uploads, amber for downloads — and supports both determinate (percentage-driven) and indeterminate (animated loop) states.

Toast with Progress

The progress indicator lives inside a toast component. The in-progress state shows a title, the animated progress bar, and a Cancel action. On completion it transitions to a success message with a Download or Confirm action.

Variants

Two color variants map to operation type. Use Download (amber) for export or file generation tasks. Use Upload (blue) for import or file transfer tasks.

Upload — Blue / Accent Brand

/**
 * Update a determinate progress bar.
 * @param {HTMLElement} fillEl  - the .cue-progress-fill element
 * @param {number}      percent - 0–100
 */
function setProgress(fillEl, percent) {
  fillEl.style.width = Math.min(100, Math.max(0, percent)) + '%';
}

// Example: poll a job and update the bar
async function trackExportJob(jobId, fillEl, onComplete) {
  const poll = setInterval(async () => {
    const res  = await fetch(`/api/jobs/${jobId}`);
    const data = await res.json();

    setProgress(fillEl, data.progress);

    if (data.progress >= 100) {
      clearInterval(poll);
      onComplete(data);
    }
  }, 500);
}

// Usage
const fill = document.querySelector('#progress-fill');
trackExportJob('abc123', fill, (data) => {
  // swap to complete toast state
  console.log('Done:', data.downloadUrl);
});

Usage Notes

Use determinate progress when you have real percentage data from the server (e.g. polling a job endpoint). Use indeterminate when a task has started but duration is unknown — this avoids a bar that appears stuck at 0%.

Always pair the progress toast with a Cancel action. On completion, swap to the success state (check icon + Download or Confirm button) rather than auto-dismissing — users should control the final action.

The progress bar does not have a visible label — the toast title provides the context. Do not add percentage text inside or adjacent to the 4px track; it would be illegible at that scale.
