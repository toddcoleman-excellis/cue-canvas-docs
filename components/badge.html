Badge

Badges communicate the status of an item at a glance. They are compact, non-interactive labels that appear inline with content — in tables, lists, cards, and alongside other UI elements. Use badges exclusively for status communication, not for navigation or decoration.

Accessibility

Attribute / PatternElementNotes
role="status" .cue-badge Communicates the badge as a status region to screen readers without interrupting the user.
aria-hidden="true" .cue-badge__icon Icon is decorative — the label text communicates the status. Never rely on icon alone.
Use a <span> .cue-badge Badges are inline, non-interactive elements. Use <span> not <div> or <button>.
Meaningful label text .cue-badge__label Label must describe the status clearly — "Fulfilled", "Processing", "On Hold" — not just "Success" or "Warning".
Color contrast All types All type/color combinations meet WCAG AA 4.5:1. Warning uses dark text (#613d00) on amber for sufficient contrast.
Don't rely on color alone All types Each badge uses a distinct icon and label text in addition to color. Always include the label.
