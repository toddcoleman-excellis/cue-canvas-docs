App Shell

The basic page layout structure for CUE Back Office. The App Shell composes the four persistent structural layers — Top Navigation, Breadcrumb, Page Header, and Footer — around the main content area. All interior pages follow this structure.

Demo

A full-fidelity rendering of the App Shell with a standard table page inside. The side-scroll handle lets you inspect the full 1440px canvas width at any viewport.

1
Top Navigation

Teal app bar fixed at the top of every page. Holds the brand logo on the left and the notifications icon + user menu on the right. Height: 55px. Background: --bg-app-top-bar (#03618d).

2
Breadcrumb Bar

White bar directly below the Top Nav. Shows the user's current location within the app hierarchy. Includes a Back link on the left separated by a pipe from the hierarchical crumb links. Height: 32px.

3
Page Header + Subnav

White header block that contains the page title row (back chevron, H1 title, subtitle, and optional client logo) and the subnav tab strip below it. The active subnav tab is indicated with an amber (#fab632) underline bar. Bottom border separates it from the content area.

4
Page Content

Main scrollable area with --app-page-bg (#f5f5f5) background. Padded 24px top/bottom and aligned to the 1200px content column. This layer holds the actual page content — typically a table, form, or module layout.

5
Footer

Teal footer matching the Top Nav color. Contains the "Powered by CUE" mark on the left, a Back to Top button on the right, a divider, and footer navigation links with copyright. Always renders at the bottom of the page.

Usage

✓ Do

  • Use this shell for every interior Back Office page — consistency is the point
  • Keep the Page Header sticky so the title and subnav remain visible while scrolling through long content
  • Always include a breadcrumb to give users a clear path back
  • Pad page content to the 1200px inner column (120px side padding at 1440px)
  • Use the subnav to break a page into logical sub-sections — keep labels short and uppercase

✗ Don't

  • Don't remove the Top Nav or Footer on interior pages — both are required chrome
  • Don't use more than one App Shell per view — the shell is singular
  • Don't replace the subnav with a secondary sidebar — keep navigation horizontal at this level
  • Don't overflow full-width content outside the 1200px column — use the padding tokens
  • Don't skip the breadcrumb on pages deeper than one level from the dashboard

Design Tokens

CSS Variable Figma Token Value Usage
--bg-app-top-bar Components/App Top Bar/bg-app-top-bar #03618d Top Nav and Footer background
--border-bottom-app-top-bar Components/App Top Bar/border-bottom-app-top-bar transparent Bottom border of Top Nav (none by default)
--bg-header Components/Header/bg-header #ffffff Page Header and Breadcrumb background
--app-page-bg System/Background/bg-page-default #f5f5f5 Main content area background
--bg-subnav-active-indicator Components/Subnav/bg-subnav-active-indicator #fab632 Amber active indicator bar on subnav tab
--content-subnav-hover-active Components/Subnav/content-subnav-hover-active #03618d Active subnav tab label color
--content-subnav-default Components/Subnav/content-subnav-default #575757 Default (inactive) subnav tab label color
--bg-footer Components/Footer/bg-footer #03618d Footer background — matches Top Nav
--content-footer Components/Footer/content-footer #ffffff Footer text and link color
--content-body-primary-brand System/Content/content-body-primary-brand #03618d Page title color
--border-level-1 System/Border/border-level-1 #ededed Separator between Breadcrumb / Header and content

Code

<!-- ① Top Navigation -->
<header class="cue-topbar">
  <div class="cue-topbar__logo">
    <!-- Brand logo (CUE glyph + wordmark) -->
  </div>
  <div class="cue-topbar__actions">
    <!-- Notifications icon -->
    <!-- User menu button -->
  </div>
</header>

<!-- ② Breadcrumb -->
<nav class="cue-breadcrumb-bar" aria-label="Breadcrumb">
  <a class="cue-breadcrumb__back">← Back</a>
  <div class="cue-breadcrumb__sep"></div>
  <ol class="cue-breadcrumb__links">
    <li><a href="/">Home</a></li>
    <li><a href="/module">Link</a></li>
    <li aria-current="page">Current</li>
  </ol>
</nav>

<!-- ③ Page Header -->
<div class="cue-page-header">
  <div class="cue-page-header__row">
    <div class="cue-page-header__left">
      <button class="cue-page-header__back" aria-label="Back">
        <span class="material-icons" aria-hidden="true">keyboard_arrow_left</span>
      </button>
      <h1 class="cue-page-header__title">Page Title</h1>
      <div class="cue-page-header__pipe" aria-hidden="true"></div>
      <p class="cue-page-header__subtitle">Page subtitle or description</p>
    </div>
    <div class="cue-page-header__logo">
      <!-- Optional client logo -->
    </div>
  </div>
  <!-- Subnav -->
  <nav class="cue-subnav" aria-label="Section navigation">
    <a class="cue-subnav__item cue-subnav__item--active">SUBNAV ITEM ACTIVE</a>
    <a class="cue-subnav__item">SUBNAV ITEM DEFAULT</a>
    <a class="cue-subnav__item">SUBNAV ITEM DEFAULT</a>
  </nav>
</div>

<!-- ④ Page Content -->
<main class="cue-page-content">
  <!-- Inner content here -->
</main>

<!-- ⑤ Footer -->
<footer class="cue-footer">
  <div class="cue-footer__top">
    <span class="cue-footer__powered">Powered by CUE</span>
    <button class="cue-footer__back-to-top">↑ Back to Top</button>
  </div>
  <hr class="cue-footer__divider" />
  <div class="cue-footer__bottom">
    <nav class="cue-footer__links">
      <a href="#">CUE Help &amp; Support</a>
      <a href="#">Terms &amp; Conditions</a>
      <a href="#">Customer Service</a>
      <a href="#">Privacy Policy</a>
      <a href="#">Terms of Sale</a>
    </nav>
    <span class="cue-footer__copy">© 2024 All Rights Reserved</span>
  </div>
</footer>
