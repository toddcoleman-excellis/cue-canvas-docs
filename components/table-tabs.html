Table Tabs

Table Tabs sit directly above a data table and allow users to switch between related datasets or views within the same surface. An amber active indicator distinguishes the selected tab; inactive tabs recede into the background bar.

Interactive Demo

Click the tabs to switch between views. The active tab shows the amber indicator and brand-colored label; inactive tabs are muted until hovered or selected.

States

Each tab has three visible states: Active (selected, white background, amber indicator, brand label), Default / Inactive (muted, subdued background, no indicator), and Hover (inactive tab transitions label to brand color on pointer entry).

Anatomy

A Table Tab is composed of a container, an optional icon, a text label, and an active indicator bar. The active indicator uses the amber accent token — the same token used across all active state indicators in the system.

Container Rounded top corners (4px), overflow hidden, padding 10px 12px 8px
Icon (optional) 18px Material Icon — matches label color per state
Label 16px — Bold (active), Regular (inactive)
Indicator 2px bar, amber (#fab632) when active, transparent when inactive

Usage Guidelines

Table Tabs are a table-specific pattern. Use them when a single module needs to surface multiple related datasets in the same layout area. They always sit flush above the table's utility bar (action bar) and share a containing card with it.

✓ Do

  • Use for switching between related, parallel datasets (Customers / Products / Orders)
  • Keep tab labels short — 1–2 words maximum
  • Always pair with the Table Utility Action Bar directly below the tabs
  • Use the optional icon when the dataset type benefits from a visual cue
  • Ensure the active tab's panel content always loads immediately — no loading placeholder for the default view

✗ Don't

  • Don't use Table Tabs for page-level navigation — use Side Navigation or Tab Navigation instead
  • Don't use more than 5 tabs; if you have more, reconsider the information architecture
  • Don't mix Table Tabs with the standard Tab Navigation component on the same surface
  • Don't use Table Tabs on non-table surfaces (cards, forms, panels)

/**
 * switchTableTab(tabId, clickedTab)
 *
 * Swaps the active state between Table Tabs and shows
 * the matching panel. Wire each tab's onclick to this function.
 *
 * @param {string}      tabId       - Matches the id of the target panel element
 * @param {HTMLElement} clickedTab  - The tab element that was clicked
 */
function switchTableTab(tabId, clickedTab) {
  const strip = clickedTab.closest('.cue-table-tabs');
  const panel = clickedTab.closest('.table-tabs-demo')
                           .querySelector('.cue-table-tab-panel');

  // Update tab states
  strip.querySelectorAll('.cue-table-tab').forEach(tab => {
    tab.classList.remove('cue-table-tab--active');
    tab.classList.add('cue-table-tab--inactive');
  });
  clickedTab.classList.remove('cue-table-tab--inactive');
  clickedTab.classList.add('cue-table-tab--active');

  // Show target panel, hide others
  panel.querySelectorAll('[id^="tab-"]').forEach(el => {
    el.style.display = 'none';
  });
  const target = panel.querySelector('#tab-' + tabId);
  if (target) target.style.display = '';
}
